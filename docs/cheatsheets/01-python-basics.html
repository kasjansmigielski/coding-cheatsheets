Python Basics ‚Äî 9 protip√≥w

1) enumerate ‚Äî numeruj elementy w pƒôtli

fruits = ["apple", "banana", "cherry"]
for i, f in enumerate(fruits, start=1):
    print(i, f)
1 apple
2 banana
3 cherry

Dlaczego warto: zamiast rƒôcznie liczyƒá indeksy.

2) zip ‚Äî paruj listy w locie

names = ["Ala", "Bartek", "Cezary"]
scores = [84, 92, 78]
for name, score in zip(names, scores):
    print(f"{name}: {score}")
Ala: 84
Bartek: 92
Cezary: 78

Protip: list(zip(names, scores)) da listƒô krotek.

4) Dictionary Comprehension ‚Äî buduj s≈Çowniki efektywnie

products = ["laptop", "mouse", "keyboard", "monitor"]
prices = [2999, 59, 199, 899]
product_dict = {product: price for product, price in zip(products, prices) if price < 1000}

Protip: Mo≈ºesz te≈º tworzyƒá s≈Çowniki z warunkami: {k: v for k, v in items if condition}.

5) f-strings ‚Äî formatuj stringi czytelnie

Dlaczego warto: Czytelniejsze i szybsze ni≈º .format() czy konkatenacja.

6) collections.Counter ‚Äî licz elementy automatycznie

from collections import Counter

7) *args i **kwargs ‚Äî elastyczne funkcje

def smart_print(*args, **kwargs):
    separator = kwargs.get('sep', ' | ')
    prefix = kwargs.get('prefix', '>>> ')
    print(prefix + separator.join(str(arg) for arg in args))

# R√≥≈ºne sposoby wywo≈Çania
smart_print("Python", "jest", "super")
smart_print("A", "B", "C", sep=" -> ", prefix="WYNIK: ")
smart_print(1, 2, 3, 4, 5, sep=", ")
>>> Python | jest | super
WYNIK: A -> B -> C
>>> 1, 2, 3, 4, 5

Dlaczego warto: Funkcje mogƒÖ przyjmowaƒá dowolnƒÖ liczbƒô argument√≥w pozycyjnych (*args) i nazwanych (**kwargs).

8) defaultdict ‚Äî s≈Çowniki z domy≈õlnymi warto≈õciami

from collections import defaultdict

# Standardowy s≈Çownik - trzeba sprawdzaƒá klucze

# if 'python' not in normal_dict:
#     normal_dict['python'] = []
# normal_dict['python'].append('awesome')

# defaultdict - automatyczne warto≈õci domy≈õlne
courses = defaultdict(list)
courses['python'].append('basics')
courses['python'].append('advanced')
courses['javascript'].append('react')

print(dict(courses))  # Konwersja do normalnego dict dla czytelno≈õci
{'python': ['basics', 'advanced'], 'javascript': ['react']}

Protip: defaultdict(int) dla licznik√≥w, defaultdict(list) dla grup, defaultdict(set) dla unikalnych warto≈õci.

9) any() i all() ‚Äî sprawdzaj warunki w kolekcjach

numbers = [2, 4, 6, 8, 10]
mixed_numbers = [1, 2, 3, 4, 5]
empty_list = []

# Czy wszystkie elementy sƒÖ parzyste?

print(f"Wszystkie parzyste w {mixed_numbers}: {all(n % 2 == 0 for n in mixed_numbers)}")

# Czy kt√≥rykolwiek element jest wiƒôkszy ni≈º 5?
print(f"Jaki≈õ > 5 w {numbers}: {any(n > 5 for n in numbers)}")
print(f"Jaki≈õ > 5 w {mixed_numbers}: {any(n > 5 for n in mixed_numbers)}")

# Edge case - pusta lista
print(f"all([]) = {all(empty_list)}")  # True (bo nie ma element√≥w fa≈Çszywych)
print(f"any([]) = {any(empty_list)}")  # False (bo nie ma element√≥w prawdziwych)
Wszystkie parzyste w [2, 4, 6, 8, 10]: True
Wszystkie parzyste w [1, 2, 3, 4, 5]: False
Jaki≈õ > 5 w [2, 4, 6, 8, 10]: True
Jaki≈õ > 5 w [1, 2, 3, 4, 5]: False
all([]) = True
any([]) = False

Data Science tip: ≈öwietne do walidacji danych - all(df['age'] > 0) sprawdzi czy ka≈ºdy wiek jest dodatni!
